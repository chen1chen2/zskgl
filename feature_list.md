# 知识库管理 Feature List

## 需求采集摘要

### 核心业务流程

- 🔴 P0 知识库列表管理：管理员在语言资产模块查看知识库卡片，按库名、知识点、概要、创建人检索，并进入知识库详情。
- 🔴 P0 新建知识库：管理员填写库名称、涉密等级，可同步上传文件或文件夹，完成知识库初始化。
- 🔴 P0 发布知识：管理员向已有知识库导入文档，设置文档密级，系统上传并进入解析流程。
- 🔴 P0 文档解析管理：管理员查看文档解析状态、摘要、语种、知识点标签，并处理预览、下载、删除、失败等结果。
- 🔴 P0 切片内容管理：管理员进入文档切片列表，新增、编辑、启停、删除、批量删除切片，并上传切片图片。
- 🔴 P0 权限与密级控制：系统按知识库角色与公开/内部/秘密密级控制查看、勾选、编辑、删除和导入导出权限。

### 用户端功能

- 🟡 P1 原译对照预览：用户在解析完成后查看原文、译文、原译对照三种预览模式。
- 🟡 P1 协作入口：用户从知识库详情页打开协作弹窗，管理或查看协作者信息。
- 🟡 P1 导入导出记录：用户查看知识库维度的导入记录、导出记录、状态、操作人和操作时间。
- 🟢 P2 全局搜索：用户在顶部搜索全部术语/语料/知识内容，快速定位语言资产。

### 管理端功能

- 🔴 P0 知识库编辑/清空/删除：管理员维护库名称、清空库内容或删除知识库，危险操作必须二次确认。
- 🟡 P1 批量密级设置：管理员对上传队列中的文件批量设置涉密等级，减少重复操作。
- 🟡 P1 操作审计与埋点：系统记录新建、上传、解析、预览、导出、删除等关键行为。
- 🟢 P2 知识质量统计：系统统计解析成功率、切片覆盖率、标签完整率、启用率和知识库健康度。

## 完整性检查

| 检查项 | 当前结论 | 补充要求 |
|-------|---------|---------|
| 新增是否有编辑/删除 | 已覆盖 | 新建库对应编辑、清空、删除；切片对应新增、编辑、删除 |
| 列表是否有筛选/排序/分页/空态 | 基本覆盖 | 文档列表和切片列表的筛选目前只是 UI，需要接入真实过滤 |
| 上传是否有失败处理 | 部分覆盖 | 有失败状态展示，缺少单文件重试、错误原因细分、网络中断恢复 |
| 删除是否有确认 | 已覆盖 | 生产版需增强为输入库名确认、请求幂等、防重复提交 |
| 权限是否前后端一致 | 待补齐 | 当前仅前端禁用态；后端必须校验角色和密级 |
| 导入是否有导出记录 | 已覆盖 | 已有导入/导出记录弹窗；需接 API、分页、筛选和导出失败重试 |
| 文档预览是否安全 | 待补齐 | 预览内容必须消毒，禁止脚本执行 |
| 大数据量是否可用 | 待补齐 | 大量知识库、文档、切片需分页、虚拟滚动、服务端搜索 |
| 审计与埋点是否完整 | 待补齐 | P0 纳入新建、上传、解析、预览、导出、删除事件 |
